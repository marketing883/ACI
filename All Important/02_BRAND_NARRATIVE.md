## MOTION & ANIMATION PRINCIPLES

### Purposeful, Not Decorative

**Use animation for:**
- Page transitions (smooth, professional)
- Hover states (subtle feedback)
- Loading states (inform, don't distract)
- Scrolling reveals (content appears as user scrolls)
- Data visualization (numbers count up)

**Don't use animation for:**
- Gratuitous effects
- Auto-playing background videos (annoying)
- Flashing/blinking elements
- Over-the-top transitions
- Anything that slows the experience

### Animation Timing
- Fast: 200ms (hover states, simple transitions)
- Medium: 400ms (page transitions, modals)
- Slow: 600ms (large movements, data visualization)

## RESPONSIVE DESIGN PRINCIPLES

### Mobile-First Mindset

**Why:** Most initial research happens on mobile

**Approach:**
1. Design for mobile (320px width)
2. Enhance for tablet (768px)
3. Optimize for desktop (1280px+)
4. Test on actual devices, not just browser tools

### Breakpoints

**Mobile:** 320px - 767px
- Single column layouts
- Hamburger navigation
- Stacked content
- Touch-friendly buttons (min 44px)

**Tablet:** 768px - 1023px
- Two-column layouts where appropriate
- Expanded navigation (still hamburger)
- Larger typography

**Desktop:** 1024px+
- Multi-column layouts
- Full navigation with dropdowns
- Maximum content width: 1440px (center-aligned)
- Generous whitespace

## FORM DESIGN PRINCIPLES

### Simple & Functional

**Contact Form:**
- Only required fields

**Lead Capture:**
- Name, email, company, role (max 4 fields)
- Explain why you're asking ("To send the whitepaper")

### Every Page Must Have

1. **Unique Title Tag** (55-60 characters)
   - Format: "[Page Topic] | ACI Infotech"
   - Example: "Data Engineering Services | ACI Infotech"

2. **Meta Description** (150-160 characters)
   - Include: Capability, proof, CTA
   - Example: "Enterprise data platforms on Databricks & Snowflake. 100+ deployments for Fortune 500. Talk to our architects."

3. **H1 Tag** (One per page)
   - Direct, keyword-rich
   - Example: "Production-Grade Data Engineering Services"

4. **Structured Data** (Schema.org)
   - Organization markup (homepage)
   - Service markup (service pages)
   - Article markup (blog posts)
   - Review markup (testimonials)

## ACCESSIBILITY REQUIREMENTS

### WCAG 2.1 AA Compliance

**Color Contrast:**
- Text: Minimum 4.5:1 contrast ratio
- Large text: Minimum 3:1
- Use contrast checker on all color combinations

**Keyboard Navigation:**
- All interactive elements accessible via Tab
- Clear focus indicators (visible border/outline)
- Logical tab order (top-to-bottom, left-to-right)
- Skip navigation link for screen readers

**Screen Reader Support:**
- Alt text on all images (descriptive, not "image of...")
- ARIA labels on interactive elements
- Proper heading hierarchy (H1 > H2 > H3)
- Form labels properly associated with inputs

**Semantic HTML:**
- Use proper elements (<button> not <div onclick>)
- Use <nav>, <main>, <article>, <aside> appropriately
- Use lists (<ul>, <ol>) for lists of items
- Use tables only for tabular data
