## SERVICE PAGE STRUCTURE (All Services)

### 1. Hero Section
- Service name
- Tagline (3-5 words)
- Headline (production-focused, specific)
- Description (2-3 sentences, technologies + outcomes)
- Key outcomes (4 bullet points)
- CTAs: "Talk to [Role] Architect" + "See [Service] Projects"

### 2. What We Build Section
- Headline: "What We Actually Build"
- Intro paragraph
- 2-3 paragraphs explaining the service reality
- Focus on production challenges, not marketing

### 3. Service Offerings Grid (4-6 Offerings per Service)
Each offering card includes:
- Title
- Description (2-3 sentences)
- Technologies list (4-6 technologies)
- Timeline (typical project duration)
- Outcomes (3 measurable outcomes)
- Use cases (3 specific scenarios)

### 4. Tech Stack Visual
- Headline: "Our [Service] Tech Stack"
- Organized by layers (3-4 layers)
- Each layer: 4-6 technologies with logos
- Highlight partnership badges where relevant

### 5. Case Studies Section
- Headline: "[Service] Projects We've Built"
- 3-4 featured case studies
- Each includes: client, challenge, solution, results, technologies
- "View All" CTA

### 6. Process Timeline
- 5 phases: Discovery → Foundation → Build → Test → Launch
- Each phase: duration, description, deliverables

### 7. Why Choose ACI Section
- Headline: "Why Choose ACI for [Service]"
- 5 reasons, each with title + description
- Focus on experience, production mindset, SLAs

### 8. FAQ Accordion
- 6-8 questions specific to the service
- Focus on: timeline, support, integration, cost optimization, training

### 9. Final CTA
- Headline with service-specific call-to-action
- Trust signals (talk to senior architects, etc.)
- Primary CTA: Schedule call
- Secondary CTA: See projects
